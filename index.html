⚠️ Hulp bij 'unauthorized_client' fout

Hoe los ik de 'unauthorized_client' fout op?

Als je de melding krijgt dat de client niet bestaat of niet is ingeschakeld voor consumenten, ondersteunt je Azure App Registratie geen persoonlijke Microsoft-accounts. Volg deze stappen:

  1. Ga naar de Azure Portal > App Registrations.
  2. Selecteer je applicatie en klik op Authentication in het menu links.
  3. Kies onder Supported account types voor:
    "Accounts in any organizational directory (Any Microsoft Entra ID tenant - Multitenant) and personal Microsoft accounts (e.g. Skype, Xbox)".
  4. Zorg dat onder de platform-configuratie voor Single-page application (SPA) de Redirect URI exact staat ingesteld op: http://localhost:5000.
  5. Kopieer de Application (client) ID en plak hem hierboven.
⚙️ Geavanceerde Instellingen

⚠️ Fout bij het inlezen van e-mails

Hoe los ik 'Failed to fetch' op?

  • Ad-blocker / Shield: Schakel je ad-blocker of 'Brave Shields' uit voor deze pagina (http://localhost:5000). Deze blokkeren vaak externe verbindingen met Google.
  • Check de URL origin: Open je de site via exact http://localhost:5000? Indien je op http://127.0.0.1:5000 of een lokaal bestand (file://) zit, blokkeert Google de aanroep wegens CORS-beveiliging.
  • Gmail API Actief? Zorg dat de Gmail API expliciet is ingeschakeld in je Google Cloud Project onder 'API Library'.
  • Gmail Sessie resetten: Probeer op de knop 'Ontkoppelen' rechtsboven te klikken om je aanmelding opnieuw te starten.

Hoe los ik deze Microsoft-fout op?

  • Consumentenondersteuning inschakelen (unauthorized_client): Deze fout betekent dat de ingevoerde Client-ID in Azure AD is geregistreerd als Single Tenant of Multitenant (alleen organisatie-accounts) in plaats van ook consumenten-accounts toe te staan.
  • Stappen in de Azure Portal:
    1. Meld je aan op de Azure Portal en ga naar App Registrations.
    2. Kies jouw geregistreerde app en klik in het linkermenu op Authentication.
    3. Scroll naar Supported account types en selecteer:
      "Accounts in any organizational directory (Any Microsoft Entra ID tenant - Multitenant) and personal Microsoft accounts (e.g. Skype, Xbox)".
    4. Zorg er ook voor dat onder de Single-page application (SPA) platform-configuratie de Redirect URI exact staat ingesteld op http://localhost:5000.
  • Standaard Client-ID restrictie: De standaard ingebouwde Client-ID werkt om veiligheidsredenen alleen op het domein waarop de app is geconfigureerd. Omdat je deze app gebruikt via een eigen domein, is het verplicht om je eigen Client-ID aan te maken en de juiste Redirect URI in te stellen.

Meld je direct af van nieuwsbrieven met de uitschrijfknoppen.

ℹ️ Hoe werkt het uitschrijven?

Wanneer je op "Uitschrijven" klikt:

  1. List-Unsubscribe: UnclutterMail stuurt een veilige, onzichtbare mail of bezoekt de automatische uitschrijflink op de achtergrond.
  2. Direct Opschonen: In Gmail-modus worden alle e-mails van die afzender direct gearchiveerd en als gelezen gemarkeerd.
  3. Fallback (🔗 Knop): Als een nieuwsbrief geen automatische header heeft, kun je de link handmatig openen.
